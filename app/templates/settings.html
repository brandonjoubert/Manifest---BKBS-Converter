Provider credentials

Must expose OpenAI-style /chat/completions (most modern providers do).

How this works

  1. Pick a preset or choose Custom.
  2. Paste your API key from that provider.
  3. Confirm base URL and model name.
  4. Save, then Test connection.
  5. Re-scan a site to extract richer BKBS entities.

Compatible providers

Any service that implements the OpenAI Chat Completions API, including:

  • xAI / Grok · OpenAI · OpenRouter · Groq
  • Together · Mistral · DeepSeek
  • Ollama / LM Studio (local)
  • Azure OpenAI, vLLM, LiteLLM, etc. via Custom URL

Security

Keys are stored in the local app database (data/bkbs.db) on this machine. They are not sent to BKBS servers (there are none). Use a secret with limited scope when possible.

Environment fallback

If no UI settings are saved, the app also reads XAI_API_KEY, OPENAI_API_KEY, LLM_API_KEY, LLM_BASE_URL, and BKBS_MODEL from .env.
